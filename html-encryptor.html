How it works
  • Your HTML file is read and encrypted with AES-256-GCM in the browser
  • Password is hashed via PBKDF2-SHA256 (200,000 iterations)
  • The output is a self-contained HTML file with a password prompt
  • Wrong password = the original content is never revealed
  • Your file never leaves your device — 100% client-side
